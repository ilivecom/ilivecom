用 Claude 给自己造一份每日 AI 简报

每天早上打开手机，刷一圈 HN、再扫两眼 TechCrunch，花了四十分钟，脑子里还是一团浆糊。AI 行业的信息密度太高，噪音也太多。我想要的其实很简单：每天一份，精、准、快，只看真正值得看的。

于是花了一个上午，给自己造了一个。

1. 让 Claude 替我去看

思路直接：用 Claude + 联网搜索，每天跑一次，把结果生成静态 HTML，推到 GitHub Pages 上。不需要服务器，不需要数据库——一个 Python 脚本，一个 HTML 模板。

信息源分三档。第一档是 OpenAI Blog、Anthropic News、Google DeepMind、Stratechery、Platformer，最可靠；第二档是 TechCrunch、Bloomberg、机器之心；第三档是 Hacker News 和 Ben's Bites，拿来补漏。筛选标准也很明确：只要技术突破、工程深度、战略信号、有分量的观点，营销稿、股价新闻、博主口水战一条都不要。

2. 多轮搜索，边找边想

普通问答是一问一答，Claude 搜一次就结束。这里用的是 Anthropic 的联网工具，让 Claude 可以连续搜好几轮——搜完第一轮，判断还缺什么，再搜，直到觉得够了才停。设了 15 轮上限，实测 5-8 轮就结束。

和单次搜索相比，内容质量差距很明显。一次性往往只抓到表层热点；多轮之后，更深的分析和更冷门但重要的内容才会浮出来。

3. 设计和主站一套

daily.hiwd.com 是 hiwd 的子站，不想让它看起来像另一个东西。直接用主站的 CSS：背景色、卡片阴影、圆角、accent 色，连标题左侧那条竖线都没变。

归档也一并做了：每天一份文件，同步更新首页，底部历史记录自动生成。域名 daily.hiwd.com，GitHub Pages 部署。

4. 细节是后来磨出来的

Claude 输出有时格式会乱，比如加粗符号和内容之间多了换行，渲染出来就断掉了。还有一个习惯——有时会在结尾自动加一行「总字数：~850 字 | 符合要求精炼程度」，像是给自己交作业。prompt 里限制了还不够，最后在后处理里加了几条正则兜底，才算稳定下来。

5. 一觉醒来简报已经在了

脚本大概 200 行 Python。麻烦的不是代码本身，是细节：怎么把筛选逻辑融进自动化流程、模板如何跟主站保持一致、归档怎么做才不重复——这些花的时间比写代码还多。

GitHub Actions 配好之后，每天早上 8 点自动触发，不用管它。

欢迎访问 daily.hiwd.com
