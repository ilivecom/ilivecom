用 Claude 给自己造一份每日 AI 简报

每天早上打开手机，刷一圈 HN、再扫两眼 TechCrunch，花了四十分钟，脑子里还是一团浆糊。AI 行业的信息密度太高，噪音也太多。我想要的其实很简单：每天一份，精、准、快，只看真正值得看的。

于是花了一个上午，给自己造了一个。

1. 想法很简单，让 Claude 替我去看

用 Claude + web_search，每天跑一次，把结果渲染成静态 HTML，推到 GitHub Pages 上。不需要服务器，不需要数据库，也不需要后台——一个 Python 脚本，一个 HTML 模板。

筛选框架直接用了 Kiara Zhang 的 prompt。信息源分三档：S 级是 OpenAI Blog、Anthropic News、Google DeepMind、Stratechery、Platformer；A 级是 TechCrunch、Bloomberg、机器之心；B 级是 Hacker News 和 Ben's Bites，拿来补漏。只要技术突破、工程深度、战略信号、有分量的观点，营销稿、股价新闻、博主口水战一条都不要。

2. Agentic loop 是关键

Anthropic 的服务端 web_search 工具（web_search_20250305）让 Claude 可以连续搜好几轮，边搜边想，直到 stop_reason 变成 end_turn——不是一次性问答，是真正的 loop。

设了 15 轮上限，实测 5-8 轮就结束。和单次问答相比，内容质量差距很明显。

3. 页面设计完全抄自 hiwd.com

不想让它看起来像另一个东西。直接把主站 CSS 移了过来：背景色、卡片阴影、圆角、accent 色，连标题左侧那条竖线都没变。

HTML 生成的 prompt 框架也是 Kiara Zhang 的，我只改了页面结构和视觉风格。archive 一并做了：每天一份 docs/archive/YYYY-MM-DD.html，同步更新 index.html，用 archive.json 维护日期索引，底部历史记录自动生成。域名 daily.hiwd.com，GitHub Pages 直接部署。

4. 细节是后来磨出来的

Claude 输出的 Markdown 有时格式会乱，比如 **\n内容\n** 这样断开的加粗。在 clean_briefing() 里一条条加正则处理：修复断行的 bold、删孤立分号、合并多余空行。

另一个问题——Claude 有时会在结尾自动加一行「总字数：~850 字 | 符合要求精炼程度」，像是给自己交作业。prompt 里限制了，我又补了条明确禁止，clean_briefing() 里再加一条兜底正则 ^总字数[：:].+$，双保险。

5. 以后每天自动跑

脚本大概 200 行 Python。麻烦的地方在细节：怎么把第三方 prompt 融进自动化流程、模板如何跟主站保持一致、archive 怎么做才不重复——这些花的时间比写代码还多。

GitHub Actions 配好之后，每天早上 CST 8 点自动触发，一觉醒来简报已经在了。

欢迎访问 daily.hiwd.com

Andrew, May 2026
